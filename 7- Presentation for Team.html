Project Charter

📋 Project Identity

  • Product: Claims of Tamriel — Cross-Platform RPG Knowledge Base & Moral Compass
  • Course: KRKE — University of Bologna, A.Y. 2025/2026

  • Start: Week 1 · End: Week 14 (May)

🎯 Project Goal

Build an OWL ontology that models the structure of disagreement in Skyrim, not game facts. Represent competing claims, rival factions, and epistemic conflict as machine-readable, SPARQL-queryable, reasoner-compatible knowledge. Serve both players and researchers.

🔭 Scope

  • 8 dispute pairs across 6 claim type categories
  • OWL 2 DL formal ontology in Turtle
  • 5 SPARQL competency questions
  • Static web interface (no backend)
  • Domain-neutral architecture

⚠ Key Risks

  • Reasoner inconsistency during axiom development
  • Lore ambiguity in source material
  • Scope creep in faction coverage
  • SPARQL query complexity vs. browser feasibility
  • Team availability near deadline

Software Requirements Specification

📌 Functional Requirements

  • FR-01: Display all 8 dispute pairs with both claim sides
  • FR-02: Show ClaimType, holder, evidence, and source URL per claim
  • FR-03: Execute SPARQL CQ1–CQ5 via competency panel
  • FR-04: Visualize moral compass and faction ideological positioning
  • FR-05: Detect neutral characters via FILTER NOT EXISTS
  • FR-06: Infer opposing holders via property chain axiom
  • FR-07: Display SKOS ClaimType hierarchy

🔧 Non-Functional Requirements

  • NFR-01: Application must run without a server or build step
  • NFR-02: Works in any modern browser
  • NFR-03: Responsive layout (desktop and mobile)
  • NFR-04: Offline capable after first load
  • NFR-05: OWL 2 DL compliant ontology
  • NFR-06: Turtle serialisation for portability

👤 User Profiles

  • Newcomer — first contact with Skyrim lore
  • World-Builder — exploring factions and narratives
  • Storyteller — writing lore-consistent stories
  • Compass-Seeker — making informed moral choices
  • Researcher — academic knowledge extraction

⛔ Constraints

  • No backend — fully static deployment
  • No claim is marked as universally true
  • No faction is treated as authoritative
  • All claims must have sourceURL from UESP Wiki
  • Every Claim must satisfy all three OWL restrictions

🗂 Class Hierarchy

  • Claim ↳ prov:Entity (16 ind.)
  • Character ↳ foaf:Person (12 ind.)
  • Faction ↳ foaf:Group (8 ind.)
  • ClaimType (SKOS, 11 concepts)
  • City ↳ schema:Place (6 ind.)
  • Item (3 ind.)
  • Race (10 ind.)

🔗 Conflict Relations

  • contradicts — SymmetricProperty. Strict logical opposition (6 pairs)
  • disputes — SymmetricProperty. Interpretive difference (2 pairs)
  • denies — IrreflexiveProperty. Directional propaganda (Talos pair only)

⛓ OWL Axioms

  • AllDisjointClasses (7 classes)
  • Claim: hasClaimHolder some Character
  • Claim: hasClaimTypeIRI exactly 1 ClaimType
  • Claim: hasDescription some xsd:string
  • PropertyChain: contradicts ∘ hasClaimHolder → hasOpposingHolder

Ontology — Reasoning

SPARQL Competency Questions

CQ1

All contradicting claim pairs with holders and types.

Technique: contradicts + FILTER(STR(?A) < STR(?B)) to deduplicate

CQ2

Faction conflict matrix — which factions oppose which.

Technique: memberOf → framesAs → contradicts chain; fully emergent

CQ3

Evidence cited by directly opposing claims.

Technique: hasEvidence on both sides of contradicts

CQ4

Neutral characters (no framesAs assertions).

Technique: FILTER NOT EXISTS — returns only Jarl Balgruuf

CQ5 — Directional Denial Chains

Traversal of asymmetric denies relation. Only the Talos pair is bidirectional — marking it as active propaganda, not mere disagreement.

CQ4 Example — Neutrality Detection

PREFIX : <http://www.semanticweb.org/skyrimontology/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?charLabel ?role
WHERE {
  ?char a :Character ;
        rdfs:label ?charLabel ;
        :hasRole ?role .
  FILTER NOT EXISTS { ?char :framesAs ?anyClaim }
}
-- Expected: Jarl Balgruuf the Greater — neutrality as structured absence

Testing Methodology & Test Case

🎯 Test Levels

  • Ontology Unit Tests — class, property, axiom
  • SPARQL Query Tests — CQ1–CQ5 output validation
  • Reasoner Consistency Tests — HermiT / Pellet
  • UI Integration Tests — view loading, data binding
  • Domain Validation — lore accuracy vs. UESP
  • Source Audit — sourceURL reachability
  • Performance — browser response under 3s

🚦 Defect Severity Levels

  • Critical — Reasoner inconsistency or SPARQL returns no results
  • Major — Incorrect claim holder or missing dispute pair
  • Minor — Wrong ClaimType assignment
  • Cosmetic — UI label formatting issue

Sample Use Case

👤 Actors & Goal

Main actor: Visitor / Player

Goal: Inspect a disputed faction choice, understand its moral and ideological consequences, and make an informed decision before committing to a side in the game.

✅ Preconditions

  • skyrim_ontology_v5_0.ttl is loaded
  • All 8 Dispute Pairs are available
  • User has access to index.html in a modern browser
  • No account required

⚔ Basic Flow

1Visitor opens index.html in the browser.
2Application loads Section IV — War Table with all 8 Dispute Pairs.
3Visitor selects: Imperial Sovereignty vs. Nordic Independence.
4Application shows both Claims with holder, ClaimType, evidence strength, conflict relation, sourceURL.
5Visitor opens World Impact Summary to understand game-world consequences.
6Visitor switches to Moral-Compass View to see faction ideological positioning.
7Optionally runs a SPARQL competency query from Section V — The Codex.
8Returns to War Table and makes an informed, evidence-grounded choice.

🔀 Alternative: Player as Guide User

If the user is actively playing the game, they can keep Claims of Tamriel open as a simultaneous guide — comparing faction positions, reading consequences, and checking disputed lore before making a permanent in-game choice.

Emergent Knowledge

Non-Obvious Discoveries

Knowledge not stated anywhere in the ontology — emergent from its structure alone.

🐉 The Dragonborn as Cosmic Arbiter

Frames claims only in existential domains: moral, cosmological, vampiric. Never territorial or political. Queryable via ClaimType filter. Positions the Dragonborn as an agent who intervenes at the level of fate, not governance.

⚔ Contested Evidence

Auriel's Bow and the Elder Scroll are each cited by directly opposing claims. The same artifact simultaneously grounds a liberation argument and a prevention argument. Contested evidence, not contested fact.

🔇 Machine-Readable Neutrality

Balgruuf's political neutrality is encoded as the absence of framesAs assertions. A single FILTER NOT EXISTS query returns exactly one result: him. Structured absence is a valid knowledge representation technique.

📣 Denial is Propagandistic

Only the Talos pair uses the directional denies relation — and both sides use it bidirectionally. This marks it as uniquely adversarial: not just logical contradiction but active propaganda. Queryable and provable.

"The ontology does not take sides.
It models the structure of taking sides."
