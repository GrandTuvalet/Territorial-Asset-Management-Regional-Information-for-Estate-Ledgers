## Project Overview

*Claims of Tamriel* is an ontology-based web application that explores contested narratives in *The Elder Scrolls V: Skyrim*. Rather than modelling objective facts, the project represents competing claims, rival factions, conflicting interpretations, and epistemic disagreement through an OWL ontology and a lightweight web interface.

The project was developed as part of the **Project Management and Software Engineering for Cultural Heritage** course and focuses on applying software engineering principles, project planning, requirements analysis, architecture design, testing, and documentation to a Digital Humanities project.

---

## Project Goal

The goal of the project is to create a structured knowledge representation system capable of:

* Modelling conflicting claims and narrative disputes
* Supporting semantic reasoning through OWL and SPARQL
* Providing an accessible interface for exploration and analysis
* Demonstrating a complete software engineering workflow from planning to delivery

## Development Approach

### Process Model

The project follows a **Waterfall development model**.

This approach was selected because:

* Requirements were defined early in the semester
* Project scope remained stable
* Deliverables were clearly specified by the course
* The project was developed within a fixed academic timeline

## Architecture Summary

The system is organised around four main components:

1. Users (Players and Researchers)
2. Interactive Web Frontend
3. Semantic Query & Reasoning Engine
4. Ontology Knowledge Base

External vocabularies such as FOAF, PROV-O, Schema.org, and SKOS are used to support interoperability and semantic alignment.
