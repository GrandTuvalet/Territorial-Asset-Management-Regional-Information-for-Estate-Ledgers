Project Overview

What We Built

Claims of Tamriel is a fully static, browser-based knowledge application powered by a formal OWL ontology — no server, no account, no build step required.

🧠
OWL 2 DL Ontology
Formal knowledge base in Turtle serialisation. 7 OWL classes, 16 claims, 8 dispute pairs, 11 SKOS ClaimTypes. Property chains, disjoint axioms, restrictions.
🌐
Interactive Web App
Single-file HTML/JS interface. War Table, Moral Compass View, Faction Cards, SPARQL Panel, Discovery Panel. Works offline after first load.
🔍
SPARQL Query Engine
5 competency questions (CQ1–CQ5) executed in-browser. Contradiction discovery, faction conflict matrix, neutral character detection, denial chains.
⚔
8 Dispute Pairs
Territorial, theological, political, moral, cosmological and supremacist conflicts. Each pair: two claims, two holders, one conflict relation, UESP-sourced evidence.
🧭
Moral Compass
Visual ideological positioning of all 8 factions across 4 alignment axes. Emergent from ontology structure — not manually drawn.
🔗
Vocabulary Alignment
Characters align to FOAF, claims to PROV-O, ClaimTypes to SKOS, places to Schema.org. Domain-neutral ontology re-usable beyond Skyrim.

🎮 Player Mode

Keep Claims of Tamriel open alongside the game. Compare faction positions, read consequences, and make informed choices before committing to a side — all evidence-grounded.

🔬 Research Mode

Enter any contested event or dispute pair and receive a structured, SPARQL-queryable, reasoner-verified knowledge view. Suitable for academic knowledge representation study.

Scope Definition

Project Scope & Boundaries

✓  IN SCOPE
  • OWL 2 DL ontology in Turtle (.ttl)
  • Exactly 8 dispute pairs, 16 individual claims
  • 7 OWL classes with axioms & restrictions
  • 11 SKOS ClaimType concepts
  • 5 SPARQL competency questions (CQ1–CQ5)
  • Static single-file HTML/JS web interface
  • Moral Compass visualisation (4 alignment axes)
  • FOAF · PROV-O · SKOS · Schema.org alignments
  • All claims sourced to UESP Wiki (sourceURL)
  • OWL 2 DL reasoner compatibility (HermiT / Pellet)
  • Property chain axiom (hasOpposingHolder)
  • GitHub-hosted deliverables & issue tracker
✕  OUT OF SCOPE
  • No backend server or database
  • No user authentication or accounts
  • No real-time game data or API integration
  • No claims beyond the 8 defined dispute pairs
  • No claim marked as universally or objectively true
  • No triplestore deployment (Fuseki / GraphDB production)
  • No native mobile application
  • No coverage of DLC factions beyond Dawnguard pair
  • No automatic lore scraping or web crawling
  • No user-generated content or annotations
  • No claim reasoning beyond OWL 2 DL profile

📐 Domain Neutrality

The ontology is designed so that its class hierarchy and property structure can be re-applied to any contested-narrative domain — history, law, politics — by swapping individuals and sourceURLs.

🎯 Success Criteria

Reasoner returns consistent; all 5 CQs return expected outputs; Moral Compass renders all 8 factions; all claims traceable to UESP; zero Critical or Major open defects at delivery.

⛔ Explicit Constraints

Fully static deployment — no build tools, no npm, no server. Works in any modern browser without installation. Offline-capable after first load. OWL 2 DL profile strictly enforced.

System Architecture

What it shows
End-to-end data flow from user interaction through the web client, SPARQL/OWL reasoning engine, and into the Turtle knowledge base. Shows optional operational infrastructure (Fuseki, Protégé, HermiT/Pellet) at the periphery.
Key layers
Users → Interactive Web Client (index.html) ⇄ Query & Reasoning Engine → Ontology Knowledge Base (.ttl) → External Vocabularies (FOAF, PROV-O, SKOS, Schema.org)
Design decision
No server-side layer by design. All reasoning executes in-browser. The same .ttl file serves both the SPARQL engine and the OWL reasoner, ensuring a single source of truth.

Ontology Design

The Knowledge Base

🗂 Class Hierarchy

  • Claim ↳ prov:Entity (16 ind.)
  • Character ↳ foaf:Person (12 ind.)
  • Faction ↳ foaf:Group (8 ind.)
  • ClaimType (SKOS, 11 concepts)
  • City ↳ schema:Place (6 ind.)
  • Item (3 ind.)
  • Race (10 ind.)

🔗 Conflict Relations

  • contradicts — SymmetricProperty · strict logical opposition (6 pairs)
  • disputes — SymmetricProperty · interpretive difference (2 pairs)
  • denies — IrreflexiveProperty · directional propaganda (Talos only)

⛓ OWL Axioms

  • AllDisjointClasses (7 classes)
  • Claim: hasClaimHolder some Character
  • Claim: hasClaimTypeIRI exactly 1 ClaimType
  • Claim: hasDescription some xsd:string
  • PropertyChain: contradicts ∘ hasClaimHolder → hasOpposingHolder

Ontology — Reasoning

SPARQL Competency Questions

CQ1

All contradicting claim pairs with holders and types.

Technique: contradicts + FILTER(STR(?A) < STR(?B)) to deduplicate symmetric pairs

CQ2

Faction conflict matrix — which factions oppose which.

Technique: memberOf → framesAs → contradicts chain; fully emergent from structure

CQ3

Evidence cited by directly opposing claims (contested artifacts).

Technique: hasEvidence on both sides of a contradicts relation

CQ4

Neutral characters — those with no framesAs assertions.

Technique: FILTER NOT EXISTS — returns only Jarl Balgruuf the Greater

CQ5 — Directional Denial Chains

Traversal of asymmetric denies relation. Only the Talos pair is bidirectional — marking it as active propaganda, not mere logical disagreement. Queryable and formally provable.

CQ4 Example — Neutrality Detection

PREFIX : <http://www.semanticweb.org/skyrimontology/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?charLabel ?role
WHERE {
  ?char a :Character ;
        rdfs:label ?charLabel ;
        :hasRole ?role .
  FILTER NOT EXISTS { ?char :framesAs ?anyClaim }
}
-- Expected: Jarl Balgruuf the Greater — neutrality as structured absence

Sample Use Case

👤 Actors & Goal

Main actor: Visitor / Player

Goal: Inspect a disputed faction choice, understand its moral and ideological consequences, and make an informed decision before committing to a side in the game.

✅ Preconditions

  • skyrim_ontology_v5_0.ttl is loaded
  • All 8 Dispute Pairs are available
  • User has access to index.html in a modern browser
  • No account or installation required

⚔ Basic Flow

1Visitor opens index.html in the browser.
2Application loads Section IV — War Table with all 8 Dispute Pairs.
3Visitor selects: Imperial Sovereignty vs. Nordic Independence.
4Application shows both Claims with holder, ClaimType, evidence strength, conflict relation, sourceURL.
5Visitor opens World Impact Summary to understand game-world consequences.
6Visitor switches to Moral-Compass View to see faction ideological positioning.
7Optionally runs SPARQL competency query CQ2 — faction conflict matrix.
8Returns to War Table and makes an informed, evidence-grounded choice.

🔀 Alternative — Player as Guide User

If the user is actively playing the game, they can keep Claims of Tamriel open as a simultaneous reference — comparing faction positions, reading consequences, and checking disputed lore before making a permanent in-game choice.

Emergent Knowledge

Non-Obvious Discoveries

Knowledge not stated anywhere in the ontology — emergent from its structure alone.

🐉 The Dragonborn as Cosmic Arbiter

Frames claims only in existential domains: moral, cosmological, vampiric. Never territorial or political. Queryable via ClaimType filter. Positions the Dragonborn as an agent who intervenes at the level of fate, not governance.

⚔ Contested Evidence

Auriel's Bow and the Elder Scroll are each cited by directly opposing claims. The same artifact simultaneously grounds a liberation argument and a prevention argument. Contested evidence, not contested fact.

🔇 Machine-Readable Neutrality

Balgruuf's political neutrality is encoded as the absence of framesAs assertions. A single FILTER NOT EXISTS query returns exactly one result: him. Structured absence is a valid knowledge representation technique.

📣 Denial is Propagandistic

Only the Talos pair uses the directional denies relation — and both sides use it bidirectionally. This marks it as uniquely adversarial: not just logical contradiction but active propaganda. Queryable and provable.

"The ontology does not take sides.
It models the structure of taking sides."
